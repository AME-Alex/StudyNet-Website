Layer 1 & 2: Physical & Data Link Layers

Layer 1 — The Physical Layer

The Physical Layer is the foundation of the entire OSI model. Its job is simple: move raw bits (1s and 0s) from one device to another through some kind of physical medium. It has no understanding of addresses, frames, or packets — it only deals with signals.

Physical Media Types

The physical medium is whatever carries the signal between devices. The three main types are:

  • Copper (Ethernet cable): Uses electrical voltage changes to represent bits. The most common type is twisted-pair cable (e.g., Cat5e, Cat6). Cheaper and easy to install, but limited in distance (~100 meters max without a repeater).
  • Fiber optic: Uses pulses of light to carry data. Much faster and capable of traveling much longer distances than copper — used in data centers and long-distance links. More expensive to install.
  • Wireless (Wi-Fi / Radio): Uses radio frequency signals transmitted through the air. Convenient but can be affected by interference, walls, and distance from the access point.
Real-world example When you plug an Ethernet cable into your laptop, the signal traveling through that cable is Layer 1 in action. The cable doesn't know what data it's carrying — it just transmits the electrical signal.

Key Layer 1 Concepts

  • Bandwidth: The maximum amount of data that can be transmitted per second (e.g., 1 Gbps, 10 Gbps). Higher bandwidth means more data can flow at once.
  • Throughput: The actual data rate achieved in real conditions — usually lower than bandwidth due to interference, congestion, or overhead.
  • Latency: The delay between sending data and it arriving at the destination. Low latency is important for things like video calls and online gaming.
  • Repeaters & Hubs: Basic Layer 1 devices. A repeater regenerates a weak signal to extend distance. A hub connects multiple devices but sends incoming data to all ports — not just the intended destination. Hubs are largely obsolete today.

Layer 2 — The Data Link Layer

The Data Link Layer takes the raw bits from Layer 1 and organizes them into meaningful chunks called frames. It's responsible for making sure data is delivered correctly within a single local network (a LAN). It also handles error detection to catch any bits that got corrupted in transit.

Frames

A frame is the Layer 2 unit of data. Think of it as an envelope: it wraps the data and includes addressing information so the network knows where it came from and where it's going — within the local network.

A typical Ethernet frame includes:

  • Destination MAC address: Where the frame is going
  • Source MAC address: Where the frame came from
  • EtherType: Tells the receiver what type of data is inside (e.g., IPv4 or IPv6)
  • Payload (data): The actual content being carried
  • CRC (Cyclic Redundancy Check): A checksum used to detect transmission errors

MAC Addresses

Every network interface card (NIC) — whether in a laptop, phone, or printer — has a unique identifier called a MAC address (Media Access Control address). This is sometimes called a "physical address" or "hardware address" because it's burned into the hardware by the manufacturer.

A MAC address is 48 bits long, typically written in hexadecimal:

Example: 00:1A:2B:3C:4D:5E

  • The first 3 bytes (00:1A:2B) identify the manufacturer (called the OUI — Organizationally Unique Identifier)
  • The last 3 bytes (3C:4D:5E) are a unique serial number assigned by the manufacturer
MAC vs. IP addresses MAC addresses are used to deliver data within a local network. IP addresses (Layer 3) are used to deliver data between networks. Both are needed to get data from your device to a website on the internet.

ARP — Address Resolution Protocol

Before a device can send a frame, it needs to know the MAC address of the destination. ARP is the protocol that handles this. Here is what happens:

  1. Device A wants to send data to Device B and knows Device B's IP address
  2. Device A broadcasts an ARP request to the entire local network: "Who has IP 192.168.1.5? Tell me your MAC."
  3. Device B sees that the IP matches its own, and replies with its MAC address
  4. Device A stores this mapping in its ARP cache for future use

Switches

A switch is the primary Layer 2 device used to connect devices on the same network. Unlike a hub (which blindly forwards data everywhere), a switch is intelligent — it keeps track of which device is connected to which port and only sends data to the intended recipient.

How a Switch Learns (MAC Address Table)

Switches build a MAC address table (also called a CAM table) by observing traffic on the network. Here is the process:

  1. When a frame arrives on a port, the switch reads the source MAC address and records which port it came from
  2. The switch then looks up the destination MAC address in its table
  3. If the destination is known, the switch forwards the frame to only that port
  4. If the destination is unknown, the switch floods the frame to all other ports (except the one it came from), and waits for a reply to learn where the destination lives
Why this matters This "learn and forward" behavior is what makes switches so much more efficient than hubs. Only the right device receives the frame, which reduces unnecessary traffic on the network.

Full-Duplex Communication

Modern switches support full-duplex — meaning a device can send and receive data at the same time. Hubs only support half-duplex, where devices take turns. Full-duplex effectively doubles the usable bandwidth and eliminates collisions.

VLANs — Virtual Local Area Networks

A VLAN allows you to logically divide a single physical switch into multiple separate networks. For example, you could put the accounting department and the engineering department on the same physical switch, but configure them to be on different VLANs so they cannot communicate directly. VLANs are a fundamental concept on the CCNA and Network+ exams.

Error Detection — CRC

Data can get corrupted during transmission — electrical interference, weak signals, or hardware issues can all flip bits. Layer 2 uses CRC (Cyclic Redundancy Check) to detect these errors.

When a frame is created, the sender runs a mathematical formula on the data and appends the result (the CRC value) to the end of the frame. When the receiver gets the frame, it runs the same formula on the data it received. If the two values match, the frame is good. If they don't match, the frame is discarded — and higher-layer protocols (like TCP at Layer 4) will handle requesting a retransmission.

Key Takeaways

Layer 1 — Physical

  • Deals with raw bits (1s and 0s)
  • Media: copper, fiber, wireless
  • Devices: repeaters, hubs
  • No concept of addressing

Layer 2 — Data Link

  • Groups bits into frames
  • Uses MAC addresses for local delivery
  • Devices: switches, network cards
  • Error detection via CRC
  • ARP resolves IP → MAC
  • VLANs logically separate networks
Exam tip On the CCNA and Network+ exams, you will frequently be asked which OSI layer a device operates at. Remember: switches = Layer 2, routers = Layer 3.
