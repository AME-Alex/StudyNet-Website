Layer 4: The Transport Layer

The transport layer provides the transparent transfer of data between hosts and is responsible for end-to-end recovery and flow control. It also deals with session multiplexing and port numbers in order to send and receive data between hosts.

Transport layer services

Transport protocols

Transmission Control Protocol (TCP): a connection-oriented transportation protocol that uses sequencing and acknowledgements to ensure reliable, ordered delivery. User Datagram Protocol (UDP): a connectionless, best-effort protocol used when low-latency is more important than reliability.

TCP vs UDP

TCP

  • Connection-oriented
  • Guaranteed transmission
  • Used by critical applications
  • Slower than UDP
  • 20 bytes header
  • Three-way handshake
  • Acknowledgement mechanism
  • Examples: HTTP, HTTPS, FTP, SMTP

UDP

  • Connectionless
  • No delivery guarantee
  • Used for real-time applications
  • Faster than TCP
  • 8 bytes header
  • No handshake mechanism
  • No acknowledgements
  • Examples: DNS, DHCP, TFTP, SNMP

Three-Way Handshake (TCP)

The three-way handshake establishes a connection between a client and server. It ensures both sides agree on initial sequence numbers and are ready to transfer data reliably.

  1. Client sends SYN (synchronize) to server.
  2. Server replies with SYN-ACK (synchronize + acknowledgement).
  3. Client sends ACK (acknowledgement) back to server.
Note: The handshake establishes sequence and acknowledgement numbers so TCP can reliably order and recover lost data.
